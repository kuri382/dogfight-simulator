3# バックグラウンドで処理させたいオブジェクトをray.Actorとして配置しつつ元のオブジェクトと同じインターフェースで透過的に操作するためのユーティリティ
4# ====== basic usage ======
5# (1) クラスclsのオブジェクトをActorとして配置する際はクラスメソッドのcreateを用いる。
6# その際、Actor生成時のoptionをdictで与える。例えば、max_concurrencyやnum_cpusなどが指定可能である。
7# なお、max_concurrencyの値1より大きくするとThreadedActorとなり複数のプロセスから同時に呼び出すことができるようになるが、
8# torchのno_gradのようにスレッドローカルな処理の場合は正しく動作しないため、そのような処理を用いる場合は1とする必要がある。
9# tro = TransparentRemoteObject.create(cls,{"max_concurrency":16,"num_cpus":1.0},*args,**kwargs)
10# (2) 「非同期でない形での」属性への操作はweakproxyと同様に、本来のclsオブジェクトと同じ書式で可能。例えば、以下のように記述できる。
11# print(tro.hoge)
12# tro.hoge = 1
13# tro.hoge += 2
14# ret = tro.fuga(tro.hoge, x = 3)
15# (3) メンバメソッドを非同期実行したい場合(バックグラウンド化する本来の目的)は、rayのActorと同じ形でremoteメソッドを用いる。
16# ref = back.async_func.remote(some_args)
17# (4) メンバ変数やメソッドの返り値は、immutableでないものはTransparentRemoteObjectとして得られる。
18# immutableな返り値はその値が直接得られる。
19# (5) 中身のオブジェクトの取得はunwrapメソッドを用いる。immutableかそうでないか不明な場合は、unwrap_if_needed(tro)を用いる。
20# actual = tro.unwrap()
21# actual = unwrap_if_needed(tro)
22# (6) 非同期実行したメソッドの実際の返り値の取得はray.getの代わりに、返り値に__call__()を呼び出すことで行う。
23# actual = ref()
24# (7) 他のプロセスとTransparentRemoteObjectを共有したい場合は、TransparentRemoteObjectInfoを介して行う。
25# TransparentRemoteObjectInfoを受け取った側は、createではなくinfoを引数として__init__を直接呼び出す。
26# <process 1>
27# tro = TransparentRemoteObject.create(cls,{"max_concurrency":16,"num_cpus":1.0},*args,**kwargs)
28# info = tro._get_info()
29# send(info) # 何らかの手段で送る。
30# <process 2>
31# info = receive() # 何らかの手段で受け取る。
32# tro = TransparentRemoteObject(info)
33#
34# ====== limitations ======
35# <general>
36# * "_get_info","unwrap","remote"という属性を持つオブジェクトを使用しないこと
37# * 特殊メソッドのうち、デスクリプタ(__get__, __set__, __delete__)は未対応
38# <numpy>
39# * __array_function__ protocolには未対応
40# * __array__()を呼ぶタイプの処理に対してはarrayのコピーが発生する
41# <torch>
42# * torch.no_grad()はスレッドローカルなコンテキストマネージャーであるため、これを使用するTransparentRemoteObjectはmax_concurrencyを1としておく必要がある。
