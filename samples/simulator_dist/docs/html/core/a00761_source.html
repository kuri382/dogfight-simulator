3// インスタンスを新規生成せず内部状態のみのシリアライゼーションを行うためのメンバ関数として
4// void serializeInternalState(AvailableArchiveTypes&,bool)が実装されている場合、
5// 通常のsave,load,serializeの代わりにそれが呼び出されるようにするためのラッパークラス。
6//
7// archive(value)の代わりにarchive(makeInternalStateSerializer(value,full))のように用いる。
8// 直接 value.serializeInternalState(archive,full) を呼び出した場合とはjson archive内の階層が異なる。
9// 前者の場合は次のjsonノードに移動して入出力が行われるのでNameValuePairとの併用が可能である。
10// 後者の場合は現在のjsonノードに続けて入出力が行われるので階層を深くしたくない場合にはそちらを用いることになる。
11// 基本的には前者(makeInternalStateSerializerを用いてラップ)を推奨する。
12//
13// valueとしてコンテナを与えた場合は、その要素型(key_type,value_type,mapped_type)について再帰的にserializeInternalStateの呼び出しを試みる。
14// このとき、serializeInternalStateを持たない型であれば通常のarchive.operator()が使用される。
15// bool型引数fullの値はコンテナの全要素について適用されるため、
16// 要素ごとに切り替えたい場合はInternalStateSerializerを要素型とする新コンテナを別途作成し、
17// 要素ごとにfullの値を設定した状態で、改めてその新コンテナに対してmakeInternalStateSerializer(value,full)を呼び出す必要がある。
18//
