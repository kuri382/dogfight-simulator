シミュレーションの処理の流れについて

シミュレーションの処理の流れ

　本項では、シミュレーションの処理の流れの概要をまとめる。一般的な gymnasium(旧 OpenAI Gym) 環境と同じく、本シミュレータの外部から見た場合に各エピソードは初期 Observation を返す reset 関数から始まり、その後は Action を入力して Observation、Reward、Terminated、Truncated、Info の 5 種類の値を返す step 関数を繰り返す。なお、本シミュレータ及び各種サンプルでは Terminated と Truncated を区別せず、旧 OpenAI gym 環境における Done と同等の扱いをしており、全ての Agent に対する Done が True となった時点でエピソードが終了する。Asset とポリシー に関するエピソード中の処理フローは下図の通りである。

シミュレーション中の処理の流れ(行動判断に関するもののみ)

　現時点での実装では、各 Agent の行動判断周期(ポリシーと Observation、Action をやりとりする周期)は 異なる値を使用してよいものとしており、1 回の step 関数で進行する tick 数は 少なくとも 1 体の Agent の行動判断が行われる時刻までであり、Agent の設定次第では一定ではなくなる。 また、図中青色で示している step 関数内部の時間進行処理は、 必ずしも全ての Asset が1[tick]ごとに処理を行うものではなく、各 Asset クラスまたはモデルごとに指定された周期で、 実行すべき時刻が来たときに実行される。

　また、Callback の各処理の実行タイミングは Callback の種類によって異なり、 下図に示すタイミングで実行される。

非周期的なイベントの発生及びイベントハンドラ

　本シミュレータでは上記で述べた周期的な処理のほかに、 突発的なイベントを発生させ、それをトリガーとしたイベントハンドラの処理を実行する機能を有している。 イベントハンドラの登録は任意のイベント名とコールバック関数オブジェクトの組を与えることで行い、 イベントを発生させる際にはイベント名とイベントハンドラに対する引数の組を与えることで行う。 イベントハンドラの引数はインターフェース共通化のため、nl::json 型の変数一つとしている。 例えば、誘導弾の命中時や戦闘機の墜落時の得点増減は Asset からイベントを発生させ Ruler が登録したイベントハンドラを呼ぶことによって実現されている。 なお、イベントハンドラの登録は Callback からのみ可能であり、 イベントの発生は Callback 、 PhysicalAsset または Controller のいずれかからのみ可能である。
