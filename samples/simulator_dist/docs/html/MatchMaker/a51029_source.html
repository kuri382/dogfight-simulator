131. MatchMaker のコンストラクタ引数について
14
15 MatchMaker クラスのコンストラクタはdict型の引数configをとる。必須のキーは以下の通り。
16 ```python
17 config={
18 "restore": None or str, #チェックポイントを読み込む場合、そのパスを指定。
19 "weight_pool": str, #重みの保存場所。<policy>-<id>.datのようにPolicy名と重み番号を示すファイル名で保存される。
20 "policy_config": { #Policyに関する設定。Policy名をキーとしたdictで与える。
21 <Policy's name>: Any, #具象クラスごとの様式に従った各Policyの設定
22 ...
23 },
24 "match_config": Any #対戦カードの生成に関する指定。具象クラスで自由に定める
25 }
26 ```

415. 重みの読み書きや初期化に関する処理の実装
42
43 MatchMaker / MatchMonitorは特定の強化学習ライブラリに依存させないために、重みを直接扱わないようにしている。
44 そのため、重みの読み書きや初期化を実際に行う処理は学習プログラム側で準備する必要がある。

501. MatchMaker.makeNextMatch のオーバーライド
51
52 対戦グループ(○○リーグのようなイメージ)を表す何らかの変数matchTypeを引数にとり
53 何らかの対戦カードを返す関数であり、各ユーザーが独自の対戦方式を実装するものである。
54
55 対戦カードはdictで表現され、場に登場するチームごとにどのPolicyのどの重みを使うかをキー"Policy"と"Weight"により指定する。
56
57 また、同名のPolicyで異なる重みを複数同時に使用したい場合等のために、それらを区別するための"Suffix"を指定可能である。
58
59 重み番号は基本的に
60 - 負数(-1): 学習中の現物
61 - 0: 学習中重みの最新のコピー(随時更新される)
62 - 自然数: 過去のある時点で保存されたコピー
63
64 とすることを想定しているが、 MatchMaker を使用する学習プログラムの書き方次第である。
65
66 また、これら以外に必要な情報があれば適宜追加してもよい。まとめると、以下のような書き方になる。
67 ```python
68 {
69 "teams": {
70 <Team's name>: {
71 "Policy": str, # teamを動かすPolicyの名前
72 "Weight": int, # teamを動かすPolicyの重み番号
73 "Suffix": str, # teamを動かすPolicyの接尾辞。同名のPolicyで異なる重みを使用する場合等に指定する。省略は不可。
74 }
75 },
76 "common": dict # teamに依存しない条件(Rule等)を記述する。派生クラスで定義する。
77 }
78 ```

802. MatchMaker.onEpisodeEnd のオーバーライド
81
82 いずれかの対戦が終わったときに呼び出し、その対戦を実施したインスタンスにおける次の対戦カードを生成して返す関数。
83
84 引数は、終了した対戦のmatchInfoと、対応する MatchMonitor.onEpisodeEnd で生成されたresultを与えるものとする。
85 また、返り値として、重み保存要否を記したdictを返す必要がある。
86 基本的には重み保存を行うPolicyのみを対象として以下のようなdictとして記述することを想定している。
87 ```python
88 {
89 <Policy's name>: {
90 "weight_id": int, # 保存先の重み番号
91 "reset": bool, # 保存後に重みをリセットするかどうか
92 }
93 }
94 ```

99
