9@par 使用時の前提
10
11-陣営名は"Blue"と"Red"を基準とする。 SimulationManager の Ruler の設定もこれに合わせること。
12-学習対象のPolicyは"Learner"とし、初期行動判断モデルは"Initial"とする。
13 SimulationManager の [AgentConfigDispatcher](#section_simulation_execution_agent_config_dispatch)
14 はサンプルに示すような形式で各Policy名に対応するaliasを登録しておくこと。
15- 一つの陣営を操作するPolicyは一種類とする。つまり、 SimulationManager の対応する Agent も一種類となる。
16- 各Policyは1体で1陣営分を動かしても、1体で1機を動かしてもよく、対戦カードにはその設定を表すbool型の"MultiPort"キーを追加している。
17- 使用する環境クラス(GymManager又はその派生クラス)をwrapEnvForTwoTeamCombatMatchMakingでラップしたものを使用し、
18 学習プログラムにおいてenv.setMatch関数を呼び出して対戦カードの反映を行うこと。
19- 対戦カードのSuffixは、学習中重みの場合は""(空文字列)、過去の重みをBlue側で使用する場合は"_Past_Blue"、過去の重みをRed側で使用する場合は"_Past_Red"とする。
20 これらのSuffixを付加した名称に対応するPolicyインスタンスを学習プログラム側で用意すること。
21
22@par ログの記録
23
24このサンプルでは対戦ログの記録についても実装例を示している。
25学習の進捗チェック用に、AlphaStar[1]で用いられているPayOffとイロレーティングを算出しており、
26それらをget_metricsでTensorboardログとして使用できる形式で出力するとともに、各エピソードの結果をcsvファイルに出力する。

28[1] Vinyals, Oriol, et al. "Grandmaster level in StarCraft II using multi-agent reinforcement learning." Nature 575.7782 (2019): 350-354.

58@par 対戦カードの生成方式
59
60- Blue側
61
62 学習中の"Learner"又は一定確率(expert_ratio)で教師役の"Expert"
63- Red側
64
65 一定エピソード数(warm_up_episodes)の経過前は常に初期行動判断モデル("Initial")とし、
66 経過後は
67 1. 学習中重みの直近のコピー
68 2. 初期行動判断モデル("Initial")
69 3. 過去の保存された重みから一様分布で選択
70
71 の3種類を4:4:2の割合で選択
72
73@par 初期状態の生成方式
74
75このクラスでは、以下のように初期状態を設定する。
76- teams[0]側について、lower_bound〜upper_boundの範囲内で、number機分の初期状態を設定する。
77- teams[1]側については、lower_boundとupper_boundをz軸を中心に点対称に反転する。
78- symmetricをTrueとした場合はteams[0]とteams[1]の各機が点対称になるように配置する。

140def keepAngle(a):
141 ret=a
142 while ret>=180.0:
143 ret-=360.0
144 while ret<=-180.0:
145 ret+=360.0
146 return ret

148def invert(src):
149 dst={
150 "pos":[-src["pos"][0],-src["pos"][1],src["pos"][2]],
151 "vel":src["vel"],
152 "heading":keepAngle(src["heading"]+180.0)
153 }
154 return dst

156def getRandomizedInitialState(teams: list[str], lower_bound: dict, upper_bound: dict, number: int, symmetric: bool):
157 """ 一定領域内でランダムな初期条件を生成する。
158
159 teams[0]側について、lower_bound〜upper_boundの範囲内で、number機分の初期状態を設定する。
160 teams[1]側については、lower_boundとupper_boundをz軸を中心に点対称に反転する。
161 symmetricをTrueとした場合はteams[0]とteams[1]の各機が点対称になるように配置する。
162 """
163 state=[{
164 "pos":[l+(h-l)*np.random.rand() for l,h in zip(lower_bound["pos"],upper_bound["pos"])],
165 "vel":lower_bound["vel"]+(upper_bound["vel"]-lower_bound["vel"])*np.random.rand(),
166 "heading": lower_bound["heading"]+(upper_bound["heading"]-lower_bound["heading"])*np.random.rand(),
167 } for i in range(number)]
168
169 if symmetric:
170 anotherState = state
171 else:
172 anotherState = [{
173 "pos":[l+(h-l)*np.random.rand() for l,h in zip(lower_bound["pos"],upper_bound["pos"])],
174 "vel":lower_bound["vel"]+(upper_bound["vel"]-lower_bound["vel"])*np.random.rand(),
175 "heading": lower_bound["heading"]+(upper_bound["heading"]-lower_bound["heading"])*np.random.rand(),
176 } for i in range(number)]
177
178 return {
179 teams[0]: {
180 "InitialState":state
181 },
182 teams[1]: {
183 "InitialState":[invert(s) for s in anotherState]
184 }
185 }

187def getFromDesiredInitialState(teams: list[str], desired: dict, lower_bound: dict, upper_bound: dict, default: list[dict], number: int):
188 """一定領域内でユーザから明示された初期条件を生成する。
189 領域外を指定された場合はクリッピングを行い、与えられた初期条件の数が不足する場合はデフォルト値を使用する。
190
191 """
192 ret={team: {"InitialState":[]} for team in teams}
193 for team in teams:
194 for i in range(number):
195 if len(desired[team])<=i:
196 s=default[i]
197 else:
198 s=desired[team][i]
199 s["pos"]=np.clip(s["pos"],lower_bound["pos"],upper_bound["pos"])
200 s["vel"]=np.clip(s["vel"],lower_bound["vel"],upper_bound["vel"])
201 lb_h=keepAngle(lower_bound["heading"])
202 ub_h=keepAngle(upper_bound["heading"])
203 s_h=keepAngle(s["heading"])
204 while ub_h < lb_h:
205 ub_h+=360.0
206 while s_h < lb_h:
207 s_h+=360.0
208 s_h=max(min(s_h,ub_h),lb_h)
209 s["heading"]=keepAngle(s_h)
210 if team==teams[1]:
211 s=invert(s)
212 ret[team]["InitialState"].append(s)
213 return ret

215def managerConfigReplacer(config,matchInfo):
216 """対戦カードを表すmatchInfoに従い、SimulationManagerのコンフィグを置き換える関数。
217 configはSimulationManager::getManagerConfig()で得られるものであり、
218 Simulationmanagerのコンストラクタに渡すenv_configのうち"Manager"キー以下の部分となる。
219 """
220 ret=copy.deepcopy(config)
221 agentConfigDispatcher=ret["AgentConfigDispatcher"]
222 assetConfigDispatcher=ret["AssetConfigDispatcher"]
223 for team, info in matchInfo.get("teams",matchInfo).items():
224 num=len(agentConfigDispatcher[team+"Agents"]["overrider"][0]["elements"])
225 #Agentが指定されていれば置換
226 if "Policy" in info and "MultiPort" in info and "Suffix" in info:
227 agentConfigDispatcher[team+"Agents"]["alias"]=info["Policy"]
228 if(info["MultiPort"]):
229 #中央集権型のAgent
230 agentConfigDispatcher[team+"Agents"]["overrider"][0]["elements"]=[
231 {"type":"direct","value":{"name":team,"port":str(i),"policy":info["Policy"]+info["Suffix"]}}
232 for i in range(num)
233 ]
234 else:
235 #SingleAssetAgent
236 agentConfigDispatcher[team+"Agents"]["overrider"][0]["elements"]=[
237 {"type":"direct","value":{"name":team+str(i+1),"policy":info["Policy"]+info["Suffix"]}}
238 for i in range(num)
239 ]
240 #初期条件が指定されていれば置換
241 if "InitialState" in info:
242 num=len(assetConfigDispatcher[team+"InitialState"]["elements"])
243 for i in range(num):
244 assetConfigDispatcher[team+"InitialState"]["elements"][i]["value"]["instanceConfig"]=info["InitialState"][i]
245 return ret

247def factoryConfigReplacer(config,matchInfo):
248 """対戦カードを表すmatchInfoに従い、SimulationManagerのFactory側のコンフィグを置き換える関数。
249 configはSimulationManager::getFactoryModelConfig(True)で得られるものであり、

346 def setAgent(self,ret,matchType,worker_index):
347 """Agentの設定を行う。このサンプルでは、以下のように設定される。
348 第一陣営(Blue)側：学習中の"Learner"又は一定確率(expert_ratio)で教師役の"Expert"
349 第二陣営(Red)側：一定エピソード数(warm_up_episodes)の経過前は常に初期行動判断モデル("Initial")とし、
350 経過後は(1)学習中重みの直近のコピー、(2)初期行動判断モデル("Initial")、(3)過去の保存された重みから一様分布で選択
351 の3種類を4:4:2の割合で選択(ただし、(3)に該当する重みが存在しない場合は(1)とする。)
352 expert_ratioとwarm_up_episodesはmatch_configで指定する。
353 また、matchType,worker_indexは特に使用しない。
354 """
355 ret[self.teams[0]]={
356 "Policy": "Learner",
357 "Weight": -1,
358 "Suffix": ""
359 }
360 expert_ratio = self.match_config.get("expert_ratio",0.0)
361 if "Expert" in self.policyInfo and expert_ratio > 0:
362 p=np.random.random()
363 if(p<expert_ratio):
364 ret[self.teams[0]]={
365 "Policy": "Expert",
366 "Weight": -1,
367 "Suffix": ""
368 }
369
370 numEpisodesForTrain=self.policyInfo["Learner"]["numEpisodesForTrain"]
371 warm_up_episodes=self.match_config.get("warm_up_episodes",1000)
372 if(numEpisodesForTrain<warm_up_episodes):
373 ret[self.teams[1]]={
374 "Policy": "Initial",
375 "Weight": -1,
376 "Suffix": ""
377 }
378 else:
379 p=np.random.random()
380 if(p<0.4):
381 ret[self.teams[1]]={
382 "Policy": "Learner",
383 "Weight": 0,
384 "Suffix": "_Past_"+self.teams[1]
385 }
386 elif(p<0.8):
387 ret[self.teams[1]]={
388 "Policy": "Initial",
389 "Weight": -1,
390 "Suffix": ""
391 }
392 else:
393 candidates=np.array(list(self.policyInfo["Learner"]["activePast"]))
394 if(len(candidates)>0):
395 wid=self.random.choice(candidates)
396 ret[self.teams[1]]={
397 "Policy": "Learner",
398 "Weight": wid,
399 "Suffix": "_Past_"+self.teams[1]
400 }
401 else:
402 ret[self.teams[1]]={
403 "Policy": "Learner",
404 "Weight": 0,
405 "Suffix": "_Past_"+self.teams[1]
406 }
407 for team in self.teams:
408 ret[team]["MultiPort"]=self.policy_config[ret[team]["Policy"]].get("multi_port",False) is True
409 return ret

411 def setInitialState(self,ret,matchType,worker_index):
412 """各Assetの初期配置の設定を行う。

424 self.match_config.get("symmetric_initial_state",True)
425 )
426 for team in self.teams:
427 ret[team]["InitialState"]=initialStates[team]["InitialState"]
428 return ret

639 def selection(self,policy):
640 """各Policyについて、場に存在するコピーの数(重みの数)を制限するために、残すものを選択する。
641 このサンプルでは、policy_configで各ポリシーの"active_limit"キーに上限値を指定することで制限を有効化し、
