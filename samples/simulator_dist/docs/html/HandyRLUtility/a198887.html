Detailed Description

HandyRL改において、リカレント部分の逐次処理を内部で実行し、
外部からは非リカレント部分と同様に全体をバッチとして一括で入力できるようにするためのラッパー
HandyRLが逐次処理を必要とする理由は、「推論を実行しない時刻」が存在したときにバッチから除外せずに、
学習時に該当時刻をマスクしてstateを更新せず次時刻に渡すことで対応されているためである。
